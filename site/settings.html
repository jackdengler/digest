Connection

Settings are saved into config.yaml in the … repository via the GitHub API, and take effect on the next digest run. Saving needs a fine-grained personal access token — it is stored only in this browser, never in the repository.

How to create the token (once, about a minute)
  1. Open github.com/settings/personal-access-tokens/new.
  2. Under Repository access choose Only select repositories and pick this repository.
  3. Under Permissions → Repository permissions set Contents: Read and write, Actions: Read and write (powers the “Run digest now” button), and Workflows: Read and write (lets you change the daily schedule).
  4. Generate the token, copy it, and paste it below.
